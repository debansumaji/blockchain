Blockchain basics

Every block remembers exactly what came before it.

A blockchain is a list of records, each one sealed with a fingerprint of the one before. Change a single letter in an old record, and every fingerprint after it stops matching. Below is a real, working chain — hashed with SHA-256, right in your browser.

The idea, in three moves

Chaining, hashing, mining.

Strip away the jargon and a blockchain is just three habits enforced by math instead of trust.

i.

Hash the contents

Every block runs its data through SHA-256, a one-way function. Same input always gives the same output — but even a one-character change scrambles it completely.

ii.

Point to the last hash

Each new block stores the previousHash of the block before it. That single pointer is what turns a pile of records into a chain.

iii.

Prove some work

Before a block is accepted, its nonce is tweaked until the hash starts with enough zeros. This "mining" makes rewriting history expensive on purpose.

The basics code

What's actually running above.

This is the real implementation behind the demo — a minimal blockchain in JavaScript, using the browser's built-in SHA-256.

blockchain.js
// A block holds data and a fingerprint (hash) of itself
class Block {
  constructor(index, timestamp, data, previousHash = '') {
    this.index = index;
    this.timestamp = timestamp;
    this.data = data;
    this.previousHash = previousHash;
    this.nonce = 0;
    this.hash = ''; // set by calculateHash()
  }

  // SHA-256 of index + timestamp + data + previousHash + nonce
  async calculateHash() {
    const input = this.index + this.timestamp +
      JSON.stringify(this.data) + this.previousHash + this.nonce;
    const buf = await crypto.subtle.digest(
      'SHA-256', new TextEncoder().encode(input)
    );
    return [...new Uint8Array(buf)]
      .map(b => b.toString(16).padStart(2, '0')).join('');
  }

  // Proof of work: keep changing the nonce until the hash
  // starts with `difficulty` zeros
  async mine(difficulty) {
    const target = '0'.repeat(difficulty);
    this.hash = await this.calculateHash();
    while (!this.hash.startsWith(target)) {
      this.nonce++;
      this.hash = await this.calculateHash();
    }
  }
}

class Blockchain {
  constructor() {
    this.chain = [new Block(0, Date.now(), 'Genesis Block', '0')];
    this.difficulty = 2;
  }

  latestBlock() { return this.chain[this.chain.length - 1]; }

  async addBlock(data) {
    const prev = this.latestBlock();
    const block = new Block(prev.index + 1, Date.now(), data, prev.hash);
    await block.mine(this.difficulty);
    this.chain.push(block);
  }

  // A chain is valid only if every hash matches its own contents
  // AND every previousHash matches the block before it
  async isValid() {
    for (let i = 1; i < this.chain.length; i++) {
      const current = this.chain[i];
      const prior = this.chain[i - 1];
      if (current.hash !== await current.calculateHash()) return false;
      if (current.previousHash !== prior.hash) return false;
    }
    return true;
  }
}
Reference

Terms worth knowing.

Hash
A fixed-length fingerprint produced from data. The same input always gives the same hash; any change to the input gives a completely different one.
Nonce
"Number used once." The value a miner changes over and over while searching for a hash that meets the difficulty target.
Difficulty
How many leading zeros a valid hash must have. Higher difficulty means more attempts, and more real computing time, per block.
Genesis block
The first block in a chain. It has no real predecessor, so its previousHash is usually just a placeholder like "0".
Immutability
Not that a chain literally can't be edited — it's that editing it is detectable, because it breaks every hash pointer downstream.
Consensus
The rule a network of computers agrees to follow — like "the longest valid chain wins" — so everyone converges on the same history without a central authority.
